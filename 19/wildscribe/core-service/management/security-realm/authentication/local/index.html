Deprecated Since 1.7.0
The security-realm configuration is deprecated and may be removed or moved in future versions.

Configuration of the local authentication mechanism.

Attributes (3)

  • allowed-users The comma separated list of users that will be accepted using the JBOSS-LOCAL-USER mechanism or '*' to accept all. If specified the default-user is always assumed allowed.
    • Attribute Value
    Type STRING
    Nillable true
    Expressions Allowed true
    Storage configuration
    Access Type read-write
    Restart Required resource-services
  • default-user The name of the default user to assume if no user specified by the remote client.
    • Attribute Value
    Type STRING
    Nillable true
    Expressions Allowed true
    Storage configuration
    Access Type read-write
    Restart Required resource-services
  • skip-group-loading Disable the loading of the users group membership information after local authentication has been used.
    • Attribute Value
    Default Value false
    Type BOOLEAN
    Nillable true
    Expressions Allowed true
    Storage configuration
    Access Type read-write
    Restart Required resource-services

Operations (2)

  • add Adds a configuration to use the JBOSS-LOCAL-USER mechanism for authentication.
    Request Parameter Type Required Expressions Allowed Default value Description
    allowed-users STRING false true The comma separated list of users that will be accepted using the JBOSS-LOCAL-USER mechanism or '*' to accept all. If specified the default-user is always assumed allowed.
    default-user STRING false true The name of the default user to assume if no user specified by the remote client.
    skip-group-loading BOOLEAN false true false Disable the loading of the users group membership information after local authentication has been used.
  • remove Removes a configuration to use JBOSS-LOCAL-USER for authentication.
